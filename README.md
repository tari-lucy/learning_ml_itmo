# MeetingScribe — AI-секретарь для совещаний

Веб-сервис для автоматической обработки аудиозаписей совещаний. На выходе — транскрипт с диаризацией спикеров и структурированное саммари.

## Как работает

Пользователь загружает аудио → FastAPI принимает задачу, резервирует кредиты на балансе и кладёт сообщение в RabbitMQ → один из двух воркеров забирает задачу и вызывает внешние ML-API → результат сохраняется в Postgres, средства списываются. Клиент опрашивает статус через `GET /predict/{task_id}`.

- **Транскрибация + диаризация:** Replicate (`thomasmol/whisper-diarization`)
- **Саммари:** vsellm (DeepSeek v3.2)

## Стек

FastAPI, Streamlit, RabbitMQ (aio-pika + pika), PostgreSQL (SQLModel), Nginx, Docker Compose

## Запуск

1. Скопировать `.env.example` → `.env` и `app/.env.example` → `app/.env`, заполнить значения.
2. Получить токены: [Replicate](https://replicate.com/account/api-tokens) и [vsellm](https://vsellm.ru).
3. Запустить:

  docker compose up --build

После старта доступны:

- Web-интерфейс (Streamlit): http://localhost:8501
- REST API (Swagger): http://localhost/docs
- Healthcheck: http://localhost/health
- RabbitMQ UI: http://localhost:15672 (логин/пароль из .env)

Функциональность

- Регистрация и авторизация (JWT)
- Просмотр и пополнение баланса в кредитах
- Загрузка аудиофайла на обработку через Web-интерфейс или REST API
- Списание кредитов с баланса при успешном выполнении задачи (резерв при приёме, подтверждение по результату, возврат при ошибке)
- Просмотр истории запросов и результатов с указанием суммы списанных кредитов
- Скачивание транскрипта и саммари
- Валидация загружаемых файлов и проверка положительного баланса перед запросом
