  # MeetingScribe — AI-секретарь для совещаний

  Веб-сервис для автоматической обработки аудиозаписей совещаний. На выходе — транскрипт с диаризацией спикеров, краткое саммари
  и структурированный протокол с реальными именами участников.

  ## Как работает

  Пользователь загружает аудио → FastAPI принимает задачу, резервирует кредиты на балансе и кладёт сообщение в RabbitMQ → один из двух воркеров забирает её и вызывает внешние ML-API → результат сохраняется в Postgres, средства списываются. Клиент опрашивает статус через `GET /predict/{task_id}`.

  После транскрибации пользователь может одной кнопкой запустить генерацию саммари и протокола. Имена спикеров (SPEAKER_00, SPEAKER_01...) можно переименовать в реальные — они автоматически применятся ко всем артефактам этого совещания.

  - **Транскрибация + диаризация:** Replicate (`thomasmol/whisper-diarization`)
  - **Саммари и протокол:** vsellm (DeepSeek v3.2)

  ## Стек

  FastAPI, Streamlit, RabbitMQ (aio-pika + pika), PostgreSQL (SQLModel), Nginx, Docker Compose

  ## Запуск

  1. Скопируйте `.env.example` → `.env` и `app/.env.example` → `app/.env`, заполните значения.
  2. Сгенерируйте `SECRET_KEY` для `app/.env` (нужен для подписи JWT-токенов):
     ```bash
     python -c "import secrets; print(secrets.token_urlsafe(32))"
  3. Получите токены: https://replicate.com/account/api-tokens и https://vsellm.ru.
  4. Запустите:
  docker compose up --build

  После старта доступны:

  - Web-интерфейс (Streamlit): http://localhost:8501
  - REST API (Swagger): http://localhost/docs
  - Healthcheck: http://localhost/health
  - RabbitMQ UI: http://localhost:15672 (логин/пароль из .env)

  Функциональность

  - Регистрация и авторизация (JWT)
  - Просмотр и пополнение баланса в кредитах
  - Загрузка аудио на обработку через Web-интерфейс или REST API
  - Транскрибация с диаризацией спикеров
  - Генерация саммари и структурированного протокола совещания одной кнопкой
  - Переименование спикеров (SPEAKER_00 → реальное имя) с автоматическим применением во всех артефактах одного совещания
  - Списание кредитов при успешном выполнении задачи (резерв при приёме, подтверждение по результату, возврат при ошибке)
  - Просмотр истории запросов и операций по балансу
  - Скачивание транскрипта, саммари и протокола в виде текстовых файлов
  - Валидация загружаемых файлов и проверка положительного баланса перед запросом
